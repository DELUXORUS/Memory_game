# Сетевая игра «Мемори» {#protocol_spec}

Текст ниже — **единый источник** описания протокола, структур данных и состава исходников (его подхватывает Doxygen как страницу **protocol_spec**). Краткий обзор проекта и запуск — на **главной странице** той же HTML-документации.

## Описание приложения

Сервер и клиент реализованы на C++ с использованием TCP-сокетов Беркли.
Сервер работает как однопроцессное однопоточное приложение с неблокирующими
сокетами и системным вызовом `poll(2)`.

Клиент подключается к серверу, передаёт имя игрока и выполняет команды
в рамках прикладного протокола.

## Протокол прикладного уровня

Протокол текстовый, основан на TCP и использует строковый разделитель `\n`.
Каждое сообщение — отдельная строка.

### Диаграмма установления соединения и обмена

Порядок сообщений после установки TCP-соединения (каждая стрелка — одна строка, завершающаяся `\n`):

```
  Клиент                          Сервер
    |                                |
    |-------- TCP connect ---------->|  accept(), Connected
    |                                |
    |---------- <имя> -------------->|
    |<--------- welcome -------------|  Waiting
    |                                |
    |-------- рекорд (опц.) --------->|
    |<----- record:<рекорд> ---------|
    |                                |
    |-------- начать ---------------->|  Playing
    |<----- symbol:<c1> -------------|
    |------ <c1> ------------------->|
    |<--------- correct -------------|
    |<----- symbol:<c2> -------------|
    |---- <c1><c2> ----------------->|
    |         ...                    |
    |<-- wrong:<длина>:<рекорд> -----|  при ошибке или сдаюсь
    |                                |
    |-------- тест (опц.) ---------->|  Testing
    |<-- symbol:... (×1000) ---------|
    |<--------- test_done -----------|  Waiting
```

### Команды клиента

- `<имя>` — регистрация игрока после подключения. Клиент сразу отправляет имя пользователя как первое сообщение (строка **без** префикса `name:`).
- `начать` — начать новую партию.
- `рекорд` — получить текущий личный рекорд.
- `сдаюсь` — завершить текущую партию.
- `тест` — запустить режим тестирования производительности.

Альтернативно клиент поддерживает английские синонимы:
`start`, `record`, `give_up`, `test`.

Во время партии клиент также отправляет **полную запомненную последовательность** одной строкой (см. логику в `src/client/client.cpp` / `src/server/server.cpp`).

Клиентский интерфейс отображает номер текущего раунда, показанную сервером последовательность,
ожидание ответа, результат каждого раунда, финальный счёт и рекорд.

### Ответы сервера

- `welcome` — сервер приветствует клиента после успешной регистрации.
- `symbol:<символ>` — сервер отправляет очередной символ для запоминания.
- `correct` — клиент правильно повторил всю текущую последовательность.
- `wrong:<длина>:<рекорд>` — клиент ошибся или сдался, сервер сообщает длину
  достигнутой последовательности и личный рекорд.
- `record:<рекорд>` — ответ на запрос рекорда.
- `test_done` — завершён режим тестирования производительности.
- `error:<описание>` — сообщение об ошибке протокола или состоянии.

### Режим тестирования

При команде `тест` (`test`) сервер отправляет клиенту 1000 символов подряд
без проверки ответов, затем отправляет `test_done`.

## Обоснование выбора структур данных

### Сервер

- `std::vector<pollfd> poll_fds_` — хранит набор дескрипторов, которые сервер
  отслеживает через `poll`. Это простой и эффективный способ управлять множественными
  неблокирующими соединениями.
- `std::map<int, Client> clients_` — позволяет быстро найти состояние клиента
  по дескриптору сокета.
- `Client` — структура, содержащая состояние пользователя:
  - `socket_fd` — дескриптор сокета;
  - `name` — имя игрока;
  - `sequence` — текущая последовательность символов для данной партии;
  - `record` — лучший результат игрока;
  - `state` — состояние протокола (`Connected`, `Waiting`, `Playing`, `Testing`);
  - `input_buffer` и `output_buffer` — буферы для неблокированной обработки данных.

### Клиент

- `std::string send_buffer_` и `recv_buffer_` — обеспечивают корректную работу
  с неблокирующими сокетами и позволяют накапливать частичные данные.
- `current_sequence_` — хранит текущую последовательность символов,
  полученную от сервера, чтобы пользователь видел полный ход игры.

## Программная реализация

- `src/server/server.cpp`, `src/server/server.hpp` — серверная логика, обработка подключений и игры.
- `src/client/client.cpp`, `src/client/client.hpp` — клиентская логика, взаимодействие с пользователем.
- `src/include/protocol.hpp` — описание прикладного протокола и вспомогательные функции.
- `src/server/server_main.cpp`, `src/client/client_main.cpp` — точки входа для запуска приложений.
